# Grasp v0.5.2 对外发布说明

日期：2026-03-23

---

## 一句话

`Grasp v0.5.2` 把项目从“会操作浏览器的工具集合”，推进成了一个更像产品的 `AI browser gateway`。

它现在不只会打开网页、点击元素，还能围绕真实网页任务给出更清晰的默认工作流：
- 进入页面
- 判断状态
- 提取内容
- 安全推进表单
- 在动态认证 workspace 里草拟、执行并验证结果

---

## 这次版本最重要的变化

### 1. 网关工具面成为默认入口

现在推荐的公开工作流是：

`entry -> inspect -> extract -> continue`

也就是说，调用方先面对的是“任务该怎么推进”，而不是一堆底层浏览器原语。

### 2. 安全真实表单任务正式进入公开能力层

现在有了一套更保守、更接近真实使用场景的表单任务流：

`form_inspect -> fill_form / set_option / set_date -> verify_form -> safe_submit`

默认行为是：
- 优先填写 `safe` 字段
- `review` 和 `sensitive` 字段显式保留
- 提交前先走 preview，而不是直接真提交

### 3. 动态认证任务流开始成立

现在 Grasp 已经能面向动态认证 workspace 提供一套更清晰的任务流：

`workspace_inspect -> select_live_item -> draft_action -> execute_action -> verify_outcome`

这意味着项目不再只适合静态页面或简单跳转，也开始能处理真实的列表、详情、草稿、发送、结果验证这类连续任务。

### 4. 发布面被收干净了

`v0.5.2` 同步完成了这几件事：
- 版本元数据统一到 `0.5.2`
- README、中文 README、docs landing、release notes 对齐
- 废弃和内部路线文档从公开文档面移除
- 发布包只保留真正需要的运行时代码和必要文件

---

## 这次版本说明了什么

`v0.5.2` 说明 Grasp 的主线已经更清楚了：

它不是一个“浏览器点击工具包”，而是在朝一个 `AI browser gateway` 演进。

它强调的是：
- 持久浏览器会话
- 带证据的页面判断
- 更安全的真实表单推进
- 动态认证任务里的草拟、执行与验证
- 接力与恢复仍然属于同一条连续工作流

---

## 这次版本不说明什么

`v0.5.2` 仍然不宣称：
- 通用绕过 CAPTCHA 或强验证
- 所有站点都能全自动完成
- 对所有复杂动态 Web App 都已经做到完美语义理解

更准确的说法是：

Grasp 现在已经能在真实网页表面上，更安全地推进任务；
但它仍然是“带验证的连续推进”，而不是“万能全自动浏览器代理”。

---

## 验证情况

当前自动化测试状态：

`174 / 174` 通过

同时，这次版本方向也已经在真实场景里做过验证，包括：
- 真实招聘表单的预填、校验、预提交路径
- 动态认证消息类 workspace 的检查、草拟、发送与结果验证路径

---

## 适合谁关注这个版本

如果你关心的是这些问题，`v0.5.2` 会比之前更值得看：
- 想给 AI 一个可持续复用的浏览器会话
- 不想每次都从“打开网页 + 找元素”重新开始
- 想让表单填写和提交前检查更安全
- 想让动态认证网站里的任务流不只是“能点”，而是“能验证”
